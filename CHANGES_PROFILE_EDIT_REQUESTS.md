# ملخص التعديلات المطلوبة

## 1) أيقونة الملف الشخصي (بدل حرف "ب")
- `client/src/components/Navigation.tsx`: عند تسجيل الدخول، تتحول الأيقونة العلوية اليمنى (كانت مربعة بحرف "ب") إلى أيقونة دائرية موحّدة لملف مستخدم (بدون صورة حقيقية) وتفتح صفحة `/profile`. للزائر غير المسجل تبقى شعار النادي كما كان (لأنه لا يوجد ملف شخصي بعد).

## 2) صفحة الملف الشخصي
- `client/src/pages/Profile.tsx`: أعيد بناؤها بالكامل لتعرض:
  - الأيقونة الرمزية + بيانات العضو (الاسم، البريد، الهاتف/واتساب، الكلية، الدائرة، التخصص، السنة الجامعية، الرقم الجامعي، تاريخ الميلاد، الخبرات الثقافية، الرقم المرجعي، الدور).
  - الفرق التي ينتمي إليها العضو (`users.getMyTeams`).
  - الأنشطة التي سجّل فيها مع حالة كل تسجيل (`users.getMyActivitySubscriptions`).
  - نموذج تعديل البيانات: أي تعديل لا يُطبَّق مباشرة، بل يُرسَل كطلب معلّق (`profileEditRequests.create`) بانتظار موافقة مسؤول. تظهر رسالة "قيد المراجعة" أثناء الانتظار، وسبب الرفض إن رُفض آخر طلب.

## 3) موافقة/رفض المسؤول
- جدول جديد `profileEditRequests` في `drizzle/schema.ts` (نفس نمط `teamJoinRequests`/`registrationRequests`: pending/approved/rejected + `rejectionReason` + `reviewedBy`/`reviewedAt`).
- دوال في `server/db.ts` لإنشاء/جلب/الموافقة/الرفض، مع قائمة بيضاء بالحقول القابلة للتعديل ذاتيًا (`PROFILE_EDITABLE_FIELDS`) بحيث لا يمكن لعضو تعديل الدور أو حالة القبول أو الرقم المرجعي عبر هذا المسار.
- راوتر جديد `profileEditRequests` في `server/routers.ts` (list/listPending/getMine/create/approve/reject) مع إشعارات داخل التطبيق للمسؤولين عند إنشاء طلب، وللعضو عند الموافقة أو الرفض.
- صفحة إدارية جديدة `client/src/pages/AdminProfileEditRequests.tsx` على المسار `/admin/profile-edit-requests` لعرض الفروقات المطلوبة لكل طلب مع زرّي قبول/رفض (الرفض يتطلب كتابة السبب)، وأُضيف رابط لها في لوحة تحكم المسؤول (`AdminDashboard.tsx`).

## 4) الفوتر
- `client/src/components/Footer.tsx`: النص الآن "جميع الحقوق محفوظة 2026 النادي الثقافي الأدبي - جامعة بوليتكنك فلسطين ،تصميم وتطوير : أحمد كامل عيده."

## ⚠️ خطوة مطلوبة قبل التشغيل
تمت إضافة جدول `profileEditRequests` إلى `drizzle/schema.ts` فقط (لا توجد قاعدة بيانات متصلة في بيئة التحرير لتوليد migration تلقائيًا). قبل النشر، شغّل:

```
pnpm db:push
```

(أو `drizzle-kit generate && drizzle-kit migrate` كما في `package.json`) لإنشاء الجدول فعليًا في قاعدة البيانات.
